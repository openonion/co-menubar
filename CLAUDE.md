## Project Overview

OO MenuBar is a native macOS menu bar application that provides quick access to ConnectOnion AI (`co ai`) from the menu bar. Built with Swift and SwiftUI, it bundles the `co` CLI inside the app for a seamless experience without requiring Python or CLI setup.

**Key Features:**
- Native macOS menu bar app (LSUIElement background app)
- Left-click icon to open chat popover
- Right-click for context menu (Start/Stop/Updates/Quit)
- Auto-update via GitHub releases
- Bundles `co` CLI binary (built with PyInstaller)
- Reads login shell environment for API keys

## Architecture

### Components

**AppDelegate.swift** (main orchestrator)
- Sets up menu bar status item with ⚡ icon
- Manages popover for chat interface
- Coordinates between agent, UI, and updater
- Handles status icon opacity (bright when running, dim when stopped)

**PopoverController.swift** (chat UI)
- NSViewController for chat popover
- Start/Stop buttons
- Text input and output display
- Manages UI state based on agent status

**Agent.swift** (process manager)
- Spawns and manages `co ai` subprocess
- Captures stdout/stderr
- Handles process lifecycle (start/stop/restart)
- State change notifications

**Updater.swift** (GitHub releases)
- Checks GitHub API for latest release
- Compares version strings
- Provides update URL for manual download

**main.swift** (entry point)
- Creates NSApplication
- Sets up AppDelegate
- Starts event loop

### Build Process

**build-app.sh** workflow:
1. Read version from `connectonion/pyproject.toml`
2. Build Swift binary with `swift build -c release`
3. Create Python venv and install connectonion + pyinstaller
4. Bundle `co` CLI as single executable with PyInstaller
5. Create .app bundle structure
6. Download and convert app icon (onion logo)
7. Generate Info.plist with version
8. Code sign with Developer ID
9. Notarize with Apple (requires credentials in .env)
10. Create distributable ZIP

## Development Commands

### Build
```bash
cd platform/oo-menubar

# Full release build (code-signed, notarized)
./build-app.sh

# Debug build (requires co CLI on PATH)
swift build
.build/debug/OOMenuBar
```

### Install
```bash
# From release build
cp -r OOMenuBar.app /Applications/
open /Applications/OOMenuBar.app

# From debug build
.build/debug/OOMenuBar
```

### Clean
```bash
rm -rf .build .build-pkg OOMenuBar.app OOMenuBar.app.zip
```

## Release Process

### Prerequisites
- macOS 12+ with Xcode Command Line Tools
- Swift 5.9+
- Python 3 with venv support
- Apple Developer ID certificate
- Apple ID with app-specific password (for notarization)

### Steps

1. **Set version** - Version is read from `connectonion/pyproject.toml`

2. **Configure credentials** - Create `.env` file:
```bash
CODESIGN_IDENTITY="Developer ID Application: Your Name (TEAMID)"
APPLE_ID="your@email.com"
APPLE_TEAM_ID="TEAMID"
APPLE_APP_PASSWORD="xxxx-xxxx-xxxx-xxxx"
```

3. **Build release**:
```bash
./build-app.sh
# Output: OOMenuBar.app.zip (signed & notarized)
```

4. **Test locally**:
```bash
cp -r OOMenuBar.app /Applications/
open /Applications/OOMenuBar.app
```

5. **Create GitHub release**:
- Tag: `v{version}` (e.g., v0.7.6)
- Upload: `OOMenuBar.app.zip`
- Release notes: changelog

### Versioning

- Version number matches `connectonion/pyproject.toml`
- Updater checks `openonion/oo-menubar` releases
- Version comparison is string-based (tag_name != CFBundleShortVersionString)

## Code Signing & Notarization

### Code Signing
- Uses Developer ID Application certificate
- Deep sign with hardened runtime: `codesign --deep --force --options runtime --sign`
- Required for Gatekeeper approval

### Notarization
- Submits to Apple via `xcrun notarytool`
- Requires Apple ID + app-specific password + team ID
- Waits for approval, then staples ticket to app
- Gracefully continues if notarization fails (code-signed build still works)

### Testing Signed App
```bash
# Verify signature
codesign -vvv --deep --strict OOMenuBar.app

# Check notarization
spctl -a -vvv -t install OOMenuBar.app

# View stapled ticket
stapler validate OOMenuBar.app
```

## Key Implementation Details

### Menu Bar Status Item
- Created with `NSStatusBar.system.statusItem(withLength: NSStatusItem.variableLength)`
- Icon: ⚡ emoji with alpha based on running state (1.0 = running, 0.55 = stopped)
- Handles both left and right clicks via `sendAction(on:)` event mask
- `LSUIElement = true` hides from dock (menu bar only)

### Process Management
- Spawns `co ai` via `Process()` pointing to bundled binary
- Uses `Pipe()` for stdout/stderr capture
- Reads shell environment with `ProcessInfo.processInfo.environment`
- Terminates via `process.terminate()` on stop

### Auto-Update
- Checks GitHub API on launch: `GET /repos/openonion/oo-menubar/releases/latest`
- Parses JSON for `tag_name` and `html_url`
- Shows "Update to X.X.X →" in context menu if newer
- Opens release page in browser (manual download)

### Popover UI
- `NSPopover` with `.transient` behavior (closes on outside click)
- Positioned below menu bar icon with `.minY` edge
- Focuses input on show with `focusInput()`
- Updates button states based on agent running status

## Troubleshooting

### Build fails: "No Developer ID certificate"
- Install Apple Developer ID certificate in Keychain
- Or set `CODESIGN_IDENTITY` in .env to your certificate name
- Or skip signing (app won't be distributable)

### Notarization fails: "Password invalid"
- Create app-specific password at appleid.apple.com
- Set `APPLE_APP_PASSWORD` in .env
- Ensure `APPLE_ID` and `APPLE_TEAM_ID` are correct

### App won't run: "Damaged or incomplete"
- Code signature may be invalid
- Re-run build-app.sh
- Or build debug version without signing: `swift build && .build/debug/OOMenuBar`

### co CLI not found in app
- PyInstaller may have failed
- Check `.build-pkg/dist/co` exists
- Rebuild: `rm -rf .build-pkg && ./build-app.sh`

### Update check fails
- GitHub API rate limit (60 req/hour unauthenticated)
- Repository name changed (check Updater.swift URL)
- No internet connection

## Design Principles

- **Simple**: One-click access to co ai from menu bar
- **Native**: Swift + AppKit for macOS-native feel
- **Bundled**: No Python or CLI setup required
- **Automatic**: Updates notify via menu bar
- **Lightweight**: Background app (LSUIElement), minimal memory
